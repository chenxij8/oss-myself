### 前端 uni-app H5 修复

**问题**：`GET http://localhost:5173/main.ts 404 (Not Found)` + `POST /api/auth/login 400 Bad Request`

**根因分析**：
1. `index.html` 中入口脚本路径写的是 `/main.ts`，但文件已迁移到 `src/main.ts`
2. 数据库中没有初始化 admin 用户，导致登录 400

**修复措施**：
1. ✅ 修正 `index.html`：`src="/main.ts"` → `src="/src/main.ts"`
2. ✅ 确认 `vite.config.ts` 中 `@` 别名正确指向 `./src`
3. ✅ 确认 `package.json` 启动命令设置 `UNI_INPUT_DIR=./src`
4. ✅ 后端添加 `DataInitializer.java`（路径：`file-server-system/.../config/DataInitializer.java`），启动时自动创建 admin/admin123 用户

**项目目录结构**（最终）：
```
file-frontend/
├── src/                    # uni-app 源码目录（通过 UNI_INPUT_DIR 指定）
│   ├── main.ts
│   ├── App.vue
│   ├── pages.json
│   ├── manifest.json
│   ├── pages/              # 页面（index, login, file, share）
│   ├── api/http.ts         # axios 封装
│   ├── stores/user.ts      # pinia store
│   └── ...
├── index.html              # Vite 入口（已修正引用 /src/main.ts）
├── vite.config.ts          # 已配置 @ 别名和 /api 代理
└── package.json            # dev:h5 使用 UNI_INPUT_DIR=./src
```

**启动方式**：
```bash
cd c:/IT/oss-myself/file-frontend
set UNI_INPUT_DIR=./src && npx uni --platform h5
```
然后访问 http://localhost:5173

**登录凭据**：admin / admin123
