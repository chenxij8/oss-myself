# 2026-04-13 工作日志

## 修复 file-frontend uni-app 依赖冲突

- **问题**：`@dcloudio/vite-plugin-uni@^0.1.0` 版本不存在导致 npm install 失败
- **根因**：package.json 中写了错误的版本号 `^0.1.0`，实际可用版本为 `3.0.0-alpha-5000720260327001`
- **解决**：
  1. 更新 package.json，所有 `@dcloudio/*` 包统一使用 `3.0.0-alpha-5000720260327001`
  2. 项目为 HBuilderX 扁平结构（manifest.json 在根目录），新版 uni CLI 默认找 `src/manifest.json`，通过 `UNI_INPUT_DIR=./` 环境变量解决
  3. npm install 成功安装 487 个包，H5 dev server 成功启动于 http://localhost:5173/
- **双端支持**：APK 打包仍需 HBuilderX 云打包，H5 可通过 `$env:UNI_INPUT_DIR="./"; npx uni --platform h5` 本地运行

## 修复 uni-app H5 路由+alias 错误

- **问题**：`GET /api/index.ts 401` + `Failed to fetch .../pages/index/index.vue`
- **根因**：`vite.config.ts` 中 `alias @` 用了 `resolve(__dirname, './')`，ESM 项目里 `__dirname` 不可用，导致路径解析错误，uni-app 框架把 `@/api` 当成 HTTP URL 发请求
- **修复**：改用 `fileURLToPath(new URL('./', import.meta.url))` 替代 `__dirname`，重启后编译成功，5174 端口正常运行
